# Developer Implementation Guide: LYW Bootstrapping and Persistence

The IPFS-Sats protocol supports two launch paths: **Zero-Cost Bootstrapping** (creator-pinned content growing toward self-sustainability) and **Funded Launch** (sufficient LYW balance to attract market-rate host payments from day one).

---

## The Two Launch Paths

### Path A: Zero-Cost Launch (Community Bootstrapping)

Content can be made persistent with zero initial LYW funding if the creator pins it on their own node (Creator Node).

The LYW is a Lightning node that generates yield by deploying liquidity — channel provisioning, liquidity leasing, and routing fee generation. This yield begins accumulating as soon as the LYW is active, even at low balance. As the content gains supporters, access payments (zaps via Lightning) flow into the LYW, growing the available balance.

**The threshold that matters for host payments is `drawdown_mode`:**
- When `drawdown_mode` is `false`: Key 3 funds host SatSwap exchange payments from the LYW's `available_sats`
- When `drawdown_mode` is `true`: host SatSwap payments are suspended; yield continues to distribute to DAO members only; the creator continues to receive income while the content's economic viability recovers

A freshly created LYW with minimal balance starts in effective drawdown state — no third-party hosts are receiving payment, so the creator's own node provides availability. As liquidity yield and supporter access payments accumulate and `available_sats` rises above the hosting cost threshold, `drawdown_mode` deactivates and host payments begin automatically.

**Viral feedback loop:** More access payments → higher `available_sats` → more hosts accept SatSwap exchanges → better availability → more access payments.

### Path B: Funded Launch

The creator funds the LYW with sufficient capital at publication time for `drawdown_mode` to be false from day one. Third-party hosts see a funded LYW and begin accepting SatSwap exchanges immediately. The content achieves redundant availability without a bootstrapping period.

The calculation below determines how much funding is required.

---

## Minimum Deposit Calculation

This function estimates the LYW funding required for the yield generated by deployed liquidity to cover market-rate host SatSwap payments indefinitely.

**Two thresholds to understand:**

| Threshold | Field | Purpose |
|---|---|---|
| Host payment threshold | `drawdown_mode` in LYW State Ledger | Below this, host SatSwap payments suspend |
| Sunset threshold | `sunset_threshold_sats` in LYW State Ledger | Below this for sustained cycles, sunset evaluation begins |

The deposit calculation targets the host payment threshold — keeping `drawdown_mode` false. The sunset threshold is a separate, lower value representing genuine economic exhaustion.

```javascript
/**
 * Calculates the minimum and recommended LYW funding for perpetual persistence.
 * Models the capital required for Lightning liquidity yield to cover
 * market-rate host SatSwap payment costs indefinitely.
 *
 * @param {number} contentSizeGB - Size of content in Gigabytes.
 * @param {number} hostCount - Desired number of redundant hosts.
 * @param {number} safetyMultiplier - Hedge against yield rate variance (default 2x).
 * @returns {Object} Minimum and recommended funding amounts with projections.
 */
function calculateMinimumLYWFunding(contentSizeGB, hostCount, safetyMultiplier = 2) {
  // Market-rate hosting cost per GB per host per month (in sats)
  // This is the rate hosts will accept via SatSwap exchanges —
  // the Host Discovery Layer's self-regulating pricing market determines
  // actual rates; this is a conservative planning estimate
  const costPerGBPerHostPerMonth = 100; // sats

  // Total monthly SatSwap payment outflow
  const monthlyHostCost = contentSizeGB * hostCount * costPerGBPerHostPerMonth;

  // Conservative Lightning liquidity yield estimate
  // Actual yield depends on LYW channel deployment strategy,
  // network conditions, and liquidity demand
  const conservativeYieldAPY = 0.10; // 10% annual
  const monthlyYieldRate = conservativeYieldAPY / 12;

  // Minimum principal: Costs / Yield Rate
  // At this balance, monthly yield exactly covers monthly host costs
  const minimumDeposit = monthlyHostCost / monthlyYieldRate;

  // Apply safety multiplier to hedge against yield rate variance
  const recommendedDeposit = minimumDeposit * safetyMultiplier;

  return {
    minimumDeposit:    Math.ceil(minimumDeposit),
    recommendedDeposit: Math.ceil(recommendedDeposit),
    monthlyHostCost,
    assumedYieldAPY:   conservativeYieldAPY,
    note: "Actual yield varies with Lightning network conditions and deployment strategy"
  };
}

// Example: 5GB content, 5 hosts
const funding = calculateMinimumLYWFunding(5, 5);
console.log(funding);

/* Output:
{
  minimumDeposit: 300000,      // 300k sats — yield exactly covers host costs
  recommendedDeposit: 600000,  // 600k sats — 2x safety margin
  monthlyHostCost: 2500,
  assumedYieldAPY: 0.10,
  note: "Actual yield varies with Lightning network conditions and deployment strategy"
}
*/
```

## Funding Guidelines by Content Type

*Assumptions: 10% APY conservative Lightning liquidity yield, 100 sats/GB/host/month hosting cost, 2× safety margin.*

| Content Type | Size | Hosts | Min Funding | Recommended | Notes |
|---|---|---|---|---|---|
| **Text Document** | 1 MB | 3 | 3,600 sats | 10,000 sats | Minimal hosting cost |
| **Image/Photo** | 5 MB | 3 | 18,000 sats | 50,000 sats | Standard redundancy |
| **Audio Track** | 10 MB | 5 | 60,000 sats | 150,000 sats | Higher availability |
| **Research Dataset** | 1 GB | 5 | 600,000 sats | 1,500,000 sats | Long-term archive |
| **Video (HD)** | 5 GB | 5 | 3,000,000 sats | 7,500,000 sats | High bandwidth needs |
| **Software Release** | 500 MB | 7 | 2,100,000 sats | 5,000,000 sats | Critical availability |

**These are planning estimates, not protocol-enforced values.** The Host Discovery Layer's pricing market sets actual host rates. A creator who funds below the minimum may still attract hosts at lower rates; a creator who funds above the recommended level builds a larger yield buffer against rate changes.

## LYW State During Bootstrapping

Key 3 monitors these LYW State Ledger fields at each distribution cycle:

```javascript
/**
 * Evaluates LYW economic health and determines host payment eligibility.
 * Called by Key 3 at each distribution_period_blocks interval.
 * @param {string} lyw_address
 * @returns {Object} Current economic state and recommended action
 */
async function evaluateLYWHealth(lyw_address) {
  const lyw = await getLYW(lyw_address);
  const ledger = lyw.state_ledger;

  const availableSats   = ledger.balance.available_sats;
  const totalSats       = ledger.balance.total_sats;
  const sunsetThreshold = ledger.balance.sunset_threshold_sats;
  const drawdownMode    = ledger.expenses.drawdown_mode;

  // Monthly yield estimate from current deployed balance
  const estimatedMonthlyYield =
    ledger.liquidity_yield.deployed_balance_sats *
    (ledger.liquidity_yield.yield_rate_ppm / 1_000_000) / 12;

  // Monthly host cost from recent expense history
  const estimatedMonthlyHostCost = ledger.expenses.host_payments_sats;

  const yieldCoversHostCosts = estimatedMonthlyYield >= estimatedMonthlyHostCost;

  return {
    drawdown_mode:           drawdownMode,
    available_sats:          availableSats,
    total_sats:              totalSats,
    yield_covers_host_costs: yieldCoversHostCosts,
    cycles_at_threshold:     ledger.balance.cycles_at_threshold,
    sunset_risk:             totalSats < sunsetThreshold,
    recommended_action: drawdownMode
      ? "Fund LYW to restore host SatSwap payments"
      : yieldCoversHostCosts
        ? "Sustaining — no action required"
        : "Consider increasing deployed balance or reducing host count"
  };
}
```

## Summary: Bootstrapping State Transitions

```
Initial state (zero or minimal funding)
  └── drawdown_mode = true
  └── Host SatSwap payments: suspended
  └── Content availability: creator node only

As LYW balance grows (yield + access payments)
  └── available_sats rises above hosting cost threshold
  └── drawdown_mode = false (automatic)
  └── Host SatSwap payments: active
  └── Content availability: redundant hosts

If LYW balance falls below threshold
  └── drawdown_mode = true (automatic)
  └── Host SatSwap payments: suspended
  └── Creator yield distribution: continues
  └── cycles_at_threshold: increments each cycle

If balance stays below sunset_threshold_sats for configured duration
  └── Sunset evaluation begins (Section 10.3)
```

The protocol never incurs debt. Expenses are never authorized beyond what `available_sats` can currently pay. Graceful degradation — not default — is the economic failure mode.
